# Origin

An Origin is the canonical source of an organisation's identity, knowledge, capability and
trust. It is the one thing everything else is a copy of.

The web taught us to build for a single consumer. The browser. The markup, the metadata and
the marketing were arranged so a person with a screen could read it. Machines got the
leftovers. A sitemap, some structured data, a scraped approximation of what the page really
meant.

That assumption no longer holds. The consumers of an organisation are now plural and unequal
in capability. People, crawlers, models and agents, each able to consume something different.
Build for one of them and the others get a degraded, drifting copy. Drift is where trust dies.
The moment the machine claim and the human page disagree, neither can be believed.

An origin fixes this by inverting the order. You do not publish a website and generate data
from it. You publish a source of truth and generate every representation from it, the website
included. The homepage is not the origin. It is one projection of it. So is the JSON, the
llms.txt and the agent manifest.

This is why an origin is structured first. Prose is a projection, not the master. Identity,
definitions, capabilities and provenance are modelled as data, versioned like software and
addressed canonically. Every representation inherits those properties for free. Consistency by
construction, provenance by default and a single place to change when the truth changes.

Heliacon is built as an origin so it can demonstrate one. The claim is simple. As more of the
internet is read by machines, the organisations that endure will be the ones with a canonical
origin, not the ones with the best looking page.
